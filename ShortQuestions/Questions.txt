Q: Difference between Runnable and Callable?
Runnable: A task returns no result. Cannot throw checked exceptions. Used with Thread or ExecutorService.submit(Runnable)
Callable: A task returns a result of type V. Can throw checked exceptions. Used with ExecutorService.submit(Callable<V>) and return a Future<V>

Q: What is the difference between t.start() and t.run()?
t.start(): starts a new thread and calls the run() method asynchronously.
t.run(): calls the run() method, no new thread is created.

Q: Which way of creating threads is better: Thread class or Runnable interface?
Using Runnable interface is better. Easier on tasks management and flexible to extend other class.

Q: Demonstrate deadlock and how to resolve it in Java code.
DDL: Two threads are trying to lock two resources in opposite order. Thread 1 locks resource 1 waits for resource 2 while thread 2 locks resource 2 and waits for resouce 1.
Solution: Always acquire lock in the same order.

Q: What is ThreadPool? How many types of ThreadPool? What is the TaskQueue in ThreadPool?
A ThreadPool is a collection of pre-created reusable threads.
There are:
1. newFixedThreadPool(int n)
2. newCachedThreadPool()
3. newSingleThreadExecutor()
4. newScheduledThreadPool(int n)

A TaskQueue is where submitted tasks wait when all threads in the pool are busy, it stores Runnable or Callable tasks until a thread becomes available.

Q: How to submit a task to ThreadPool?
We can use ExecutorService to submit tasks:
ExecutorService executor = Executors.newFixedThreadPool(4);
executor.execute(() -> System.out.println("Hello")); // Runnable no result
Future<Integer> future = executor.submit(() -> 42); // Callable return result

Q: What is the advantage of ThreadPool?
1. Avoid frequent thread creation. 2. Limit the number of active threads. 3. Support queuing, scheduling, cancellation

Q: Difference between shutdown() and shutdownNow() methods of executor
shutdown(): Stops accepting new tasks, lets submitted tasks finish
shutdownNow(): Attempts to stop all tasks immedeiatly, interrupts running threads, returns pending tasks.
