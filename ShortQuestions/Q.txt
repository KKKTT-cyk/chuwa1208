2. Explain tight coupling vs loose coupling and what does Spring IOC do?
Tight Coupling: A class has a concret implementation of another class.
Loose Coupling: A class depends on an abstraction such as Interface.
Spring IOC: Inversion of Control, it's Bean Factory that manage the program lifecycle and dependency injection.


3. What is MVC pattern?
Model: Data object that can be used in database
View: Front-end, user can see it.
Controller: API methods.

4. What is Front-Controller?
A design pattern where a single central controller handles all incoming requests and delegates them to appropriate handlers.

5. Explain DispatcherServlet and how it works.
DispatcherServlet is the Front Controller of Spring MVC.
It receives all incoming HTTP requests, corrdinates the entire request-response workflow, and delegates work to the right components.
